# AI Benchmark Platform 使用说明

## 项目结构

```
ai-benchmark-platform/
├── src/                    # 前端 React + TypeScript + Ant Design
├── backend/                # 后端 FastAPI 服务
│   ├── start.sh            # 后端启动脚本
│   ├── requirements.txt    # Python 依赖
│   ├── ssh-targets.example.json  # SSH 目标配置示例（复制后改路径，勿提交密钥）
│   └── app/
│       ├── main.py         # FastAPI 入口
│       ├── remote_worker.py # SSH 目标机执行 test-cases：`python -m backend.app.remote_worker`
│       ├── routers/        # API 路由
│       ├── schemas/        # 数据模型
│       └── services/       # 业务逻辑（含 OpenCompass 集成）
├── opencompass/            # OpenCompass 评测框架（git clone）
├── vite.config.ts          # 前端构建配置 + API 代理
└── package.json
```

---

## 一、环境准备

### 1.1 系统要求

- Python >= 3.10
- Node.js >= 18
- NVIDIA GPU + CUDA 驱动
- nvidia-smi 可用

### 1.2 安装 OpenCompass 依赖

```bash
cd ai-benchmark-platform/opencompass
pip install -e .
# 或手动安装运行时依赖
pip install -r requirements/runtime.txt
```

如需使用 vLLM 后端：

```bash
pip install vllm
```

如需使用 LMDeploy 后端：

```bash
pip install lmdeploy
```

### 1.3 安装后端依赖

```bash
pip install -r backend/requirements.txt
```

### 1.4 安装前端依赖

```bash
npm install
```

## 二、启动服务

需要同时启动后端和前端，使用两个终端。

### 2.1 启动后端（终端 1）

```bash
# 默认启动
./backend/start.sh

# 开发模式（代码变更自动重载）
./backend/start.sh --reload
```

后端默认监听 `0.0.0.0:8711`。

可通过环境变量自定义：

```bash
PORT=9000 ./backend/start.sh           # 修改端口
OPENCOMPASS_ROOT=/other/path ./backend/start.sh  # 指定 OpenCompass 路径
```

验证后端启动成功：

```bash
curl http://127.0.0.1:8711/health
# 预期输出: {"status":"ok"}

curl http://127.0.0.1:8711/api/v1/system/gpu-info
# 预期输出: GPU 硬件信息 JSON
```

### 2.2 启动前端（终端 2）

```bash
npm run dev
```

前端默认运行在 `http://localhost:5173`，已配置代理将 `/api/v1/*` 请求转发到后端 `127.0.0.1:8711`。

> 如果修改了后端端口，需同步修改 `vite.config.ts` 中的 proxy target。

## 三、加速卡评测

在前端页面进入 **加速卡任务中心**，切换到 **"OpenCompass 评测"** Tab。

### 3.1 通过前端创建评测任务

1. 点击 **"新建评测任务"**
2. 填写配置：
   - **执行位置**：**本机**（运行后端的机器）或 **SSH 远程**（需先配置 `BENCHMARK_SSH_TARGETS`，见下文 §3.4）
   - **任务名称**：自定义标识
   - **模型路径**：HuggingFace 模型 ID 或本地绝对路径（如 `/path/to/Qwen3.5-35B-A3B`）；若选 SSH，请填 **目标机上** 可访问的路径
   - **推理后端**：`huggingface` / `vllm` / `lmdeploy`
   - **数据集**：勾选需要评测的数据集（gsm8k, mmlu, ceval, humaneval 等）
   - **GPU 数量**：1~8
   - **Batch Size**：默认 8
   - **Max Output Length**：默认 256
3. 提交后任务自动开始运行
4. 任务列表中可查看状态、点击查看详情（实时日志、GPU 指标、评测分数）

### 3.2 通过 API 创建评测任务

```bash
curl -X POST http://127.0.0.1:8711/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Qwen3.5-35B vLLM 评测",
    "model_path": "/path/to/Qwen3.5-35B-A3B",
    "datasets": ["gsm8k"],
    "backend": "vllm",
    "num_gpus": 1,
    "batch_size": 8,
    "max_out_len": 512
  }'
```

### 3.3 查看任务状态

```bash
# 任务列表
curl http://127.0.0.1:8711/api/v1/tasks

# 单个任务详情
curl http://127.0.0.1:8711/api/v1/tasks/{task_id}

# 任务日志
curl http://127.0.0.1:8711/api/v1/tasks/{task_id}/logs

# 取消任务
curl -X POST http://127.0.0.1:8711/api/v1/tasks/{task_id}/cancel

# 删除任务
curl -X DELETE http://127.0.0.1:8711/api/v1/tasks/{task_id}
```

### 3.4 SSH 远程执行

在 **加速卡任务中心** 中，OpenCompass 评测与各 **测试用例 Tab** 的任务均可选择在本机或通过 **SSH** 在远程 GPU 机器上执行。控制端（运行 FastAPI 的电脑）只负责下发任务与展示日志/结果，**不在浏览器里暴露 SSH 密钥**。

#### 前置条件

- **控制端**：已安装 OpenSSH 客户端（`ssh`、`scp` 可用）；能通过网络登录目标机（密钥登录推荐）。
- **目标机**：已克隆与本项目一致的代码树；已安装 `backend/requirements.txt`；OpenCompass 任务需在目标机安装好 `opencompass`（与 §1.2 相同）。
- **OpenCompass 评测（SSH）**：目标机上 `project_root` 与 `opencompass_root` 路径正确，且 `model_path`、数据集缓存均在 **目标机** 可读。
- **加速卡测试任务（SSH）**：目标机在 `project_root` 下能执行  
  `python3 -m backend.app.remote_worker test-case <请求文件>`（即 `backend` 包可导入、各 test runner 可用）。

#### 配置 SSH 目标

在启动后端的 shell 中设置其一即可：

1. **环境变量 `BENCHMARK_SSH_TARGETS`**：值为 **JSON 数组** 字符串（单行或导出前写入文件再 `export $(cat file)`）。
2. **环境变量 `BENCHMARK_SSH_TARGETS_FILE`**：指向 JSON 文件路径（便于维护多行格式）。

字段说明（每项一条目标）：

| 字段 | 说明 |
|------|------|
| `id` | 唯一标识，前端与 API 里 `target_id` 引用此项 |
| `host` / `port` / `user` | SSH 主机、端口、用户名 |
| `identity_file` | （可选）**控制端**上私钥路径；不配则使用默认 ssh-agent/密码策略（无界面场景建议配密钥） |
| `project_root` | 目标机上本仓库根目录的**绝对路径**（其下应有 `backend/`） |
| `opencompass_root` | （可选）目标机 OpenCompass 根目录；默认 `{project_root}/opencompass` |
| `python_executable` | （可选）默认 `python3` |

完整示例见仓库内 **`backend/ssh-targets.example.json`**。配置改好后需 **重新启动后端** 或重新加载进程，使环境变量生效。

#### 验证

```bash
# 列出已加载的 SSH 目标（无敏感字段）
curl http://127.0.0.1:8711/api/v1/system/ssh-targets

# 本机 GPU
curl http://127.0.0.1:8711/api/v1/system/gpu-info

# 指定目标机在远程执行 nvidia-smi（id 与 JSON 中一致）
curl "http://127.0.0.1:8711/api/v1/system/gpu-info?target_id=lab-gpu-1"
```

#### 请求体中的 `execution` 字段

创建 **OpenCompass 任务** 或 **test-cases 任务** 时均可附加（省略则等价本机）：

```json
"execution": {
  "mode": "local",
  "target_id": null
}
```

远程示例：`"mode": "ssh"`，且 `"target_id"` 为配置中的 `id`。

通过 API 创建 OpenCompass 远程任务示例：

```bash
curl -X POST http://127.0.0.1:8711/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "name": "远程 vLLM 评测",
    "model_path": "/remote/path/to/model",
    "datasets": ["gsm8k"],
    "backend": "vllm",
    "num_gpus": 1,
    "batch_size": 8,
    "max_out_len": 512,
    "execution": { "mode": "ssh", "target_id": "lab-gpu-1" }
  }'
```

#### 行为摘要

- **OpenCompass（SSH）**：控制端生成本地配置后上传到目标机工作目录，在目标机执行 `run.py`，日志回传至任务详情；结束后将远程输出目录同步回本地 `outputs/accelerator/{task_id}` 再解析分数。
- **test-cases（SSH）**：控制端将任务描述 `scp` 至目标机 `.benchmark_staging/`，远程执行 `remote_worker` 写回结果 JSON 后拉回。

## 五、端口冲突处理

如果启动后端报 `Address already in use`：

```bash
# 查找占用端口的进程
ss -tlnp | grep 8711

# 杀掉占用进程
kill -9 $(lsof -ti:8711)

# 重新启动
./backend/start.sh --reload
```

## 七、数据集管理工具

项目提供三个 npm 脚本用于管理评测数据集，数据来源包括 HuggingFace 开源数据集和本地自建数据集。

### 7.1 全量同步数据集

```bash
npm run sync:datasets
```

从 HuggingFace 下载完整数据集（parquet 文件）+ 预览样本，同时生成自建数据集的样本文件。输出：

- `public/datasets/catalog.json` — 数据集总目录
- `public/datasets/open-source/<id>/sample.jsonl` — 每个开源数据集的预览样本
- `public/datasets/open-source/<id>/raw/parquet/*.parquet` — 完整数据文件
- `public/datasets/self-built/<id>/sample.jsonl` — 自建数据集样本

> 全量下载耗时较长（取决于网络和数据集大小），首次运行建议先用预览模式。

### 7.2 仅同步预览样本

```bash
npm run sync:datasets:preview
```

仅从 HuggingFace Datasets Server 获取前 200 行预览数据，不下载完整 parquet 文件。适合快速初始化 catalog 或网络条件不佳时使用。

### 7.3 Parquet/Arrow 转 test.jsonl

```bash
npm run parquet:to-test-jsonl
```

将全量同步后 `public/datasets/open-source/*/raw/` 目录下的 parquet 或 HuggingFace Arrow 文件转换为 `test.jsonl` 格式（每行一个 JSON 对象）。如数据集包含图片，图片会被提取到 `<dataset>/image/` 目录，jsonl 中记录相对路径。

支持 `--dry-run` 参数预览操作而不实际写入：

```bash
npm run parquet:to-test-jsonl -- --dry-run
```

### 7.4 依赖说明

这三个脚本使用系统 Python3 运行，需要以下 Python 包：

| 包名 | 用途 | 必需 |
|------|------|------|
| pyarrow | 读取 parquet 文件 | parquet:to-test-jsonl 必需 |
| datasets | HuggingFace 数据集加载（备用下载方式） | 可选 |
| huggingface_hub | 门控数据集下载（如 ImageNet） | 可选 |

如需访问门控数据集（如 ImageNet-1K），需设置 HuggingFace Token：

```bash
export HF_TOKEN=hf_your_token_here
npm run sync:datasets
```

## 八、注意事项

1. **任务存储**：当前使用内存存储，后端重启后任务记录会丢失
2. **模型路径**：本机模式需确保 **API 进程** 能访问模型；**SSH 模式**需确保 **目标机** 能访问对应路径
3. **数据集下载**：首次运行某个数据集时，OpenCompass 会自动从 HuggingFace 下载，需要网络连接（下载发生在实际执行评测的机器上）
4. **GPU 占用**：评测任务会占用 GPU 显存；SSH 模式下占用的是 **目标机** GPU
5. **并发限制**：当前配置为单 worker，任务串行执行
6. **SSH 安全**：私钥路径仅配置在运行后端的环境中，勿将密钥或含密钥的 JSON 提交到 Git；生产环境建议使用专用部署账号与最小权限
7. **SSH 依赖**：控制端必须能调用系统 `ssh`/`scp`；若目标机 `StrictHostKeyChecking` 首次连接，需保证网络与主机指纹策略可接受（当前后端使用 `accept-new` 等常用选项）
