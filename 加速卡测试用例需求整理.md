# 网安加速卡测试用例需求整理（v20260317）

本文基于 `网安-加速卡测试用例20260317.docx` 的已转写内容整理，目标是把原始测试说明转成便于执行与评审的需求清单。

## 1. 目标与范围

- 目标：形成可重复执行的加速卡评测规范，覆盖基础能力、训练、推理、精度、生态兼容、虚拟化与视频编解码。
- 范围：单机多卡服务器场景，包含 Docker 环境、驱动/计算引擎、主流训练推理框架及工具链验证。
- 产出：每个用例必须有可量化指标、可复现实验步骤、可审计日志与结果记录。

## 2. 通用测试前提（所有用例共用）

- 服务器供电正常，BIOS 为稳定商用版本。
- 操作系统、驱动、计算引擎、Docker 环境安装正确并运行正常。
- AI 芯片在位且健康状态正常。
- 对应测试工具（厂商带宽/算力工具、训练框架、推理框架、精度评测工具等）已安装完成。
- 测试过程需保留命令、日志、结果文件，便于复核。

## 3. 测试域与验收需求

## 3.1 AI 芯片基础能力

1. AI 芯片间通信带宽测试  
   - 目的：验证单机内芯片间双向通信带宽。  
   - 验收：去除对角线后，对其余双向带宽求平均，输出 GB/s。

2. AI 芯片显存带宽测试  
   - 目的：验证每张卡显存带宽。  
   - 验收：记录每张卡显存带宽（GB/s）。

3. AI 芯片基础算力测试  
   - 目的：验证不同精度算力能力。  
   - 验收：记录 FP64/FP32/TF32/BF16/FP16/INT8 下的算力（TFLOPS/TOPS）。

4. AI 芯片功耗测试  
   - 目的：验证满载稳定功耗。  
   - 验收：记录满载状态下 10 分钟平均功耗。

5. 高负载稳定性测试  
   - 目的：验证 2*24h 持续高压稳定性。  
   - 验收：压力程序全程正常，无告警、无重启、无宕机、无关键日志异常。

## 3.2 模型训练性能

1. ResNet50 训练性能（单卡/8卡）  
   - 框架与数据：PyTorch 2.7.1 + ImageNet2012。  
   - 精度模式：FP32、FP16 混合精度。  
   - 验收：记录训练吞吐 `imgs/s`。

2. YOLOv5-L 训练性能  
   - 验收：记录训练吞吐 `imgs/s`。

3. Llama3-8B 预训练性能  
   - 验收：记录 `TGS = global_bs * seqlen / gpus / times`。

4. Qwen3-8B 预训练性能  
   - 验收：记录 `TGS = global_bs * seqlen / gpus / times`。

## 3.3 模型推理性能

1. DeepSeek-V3-0324 推理性能  
   - 验收：在满足 SLO（TTFT/TPOT 门限）下，记录最高并发时 Output Throughput、TTFT、TPOT。

2. DeepSeek R1-Channel-INT8 推理性能  
   - 验收：同上，记录 SLO 约束下吞吐与时延指标。

3. DeepSeek-R1-Distill-Qwen-32B（变长）  
   - 验收：记录 `total_token_throughput`。

4. Qwen3-32B（定长）  
   - 验收：记录并发数、输入/输出长度、TTFT、TPOT、生成吞吐等。

5. GLM4.7-W8A8  
   - 验收：记录并发、输入/输出长度、TTFT、TPOT、生成吞吐等。

6. YAYI  
   - 验收：记录 `total_token_throughput`。

7. Bert 推理性能  
   - 验收：记录平均处理帧率 `fps`。

## 3.4 模型精度

1. OpenCompass 精度测试（DeepSeek-R1-671B INT8）  
   - 验收：记录 gsm8k 准确性指标。

2. EvalScope 精度测试（DeepSeek-R1-671B INT4）  
   - 验收：记录 gsm8k 准确性指标。

## 3.5 生态兼容性

1. Verl 训练框架兼容  
   - Mixtral-8x7B MOE 微调兼容性。  
   - Qwen2.5-0.5B-GPRO 训练兼容性。  
   - 验收：训练可正常启动，Loss 正常下降。

2. RAG 领域模型兼容  
   - bge-large-zh-v1.5（Embedding）。  
   - bge-reranker-large（Rerank）。  
   - 验收：推理流程可启动，输出向量/重排序结果正确。

3. 国产芯片 CUDA 迁移适配能力  
   - 验收：样例可编译并成功执行，成功样例不少于 80 个。

4. 基于 Kubernetes 的 GPU/DCU 虚拟化  
   - 验收：直通与虚拟化场景下资源分配显示正确。

## 3.6 视频编解码能力

1. Heepstream 插件测试  
   - 验收：解码出现 `finish`，输出 `temp.yuv`；编码生成 `output.h264`。

2. CodeTest 工具测试  
   - 验收：输出各测试用例结果。

3. CodecBenchmark 测试  
   - 验收：可输出设定核数下的性能结果。

## 4. 指标与结果记录要求

- 性能类：吞吐（imgs/s、token/s）、时延（TTFT、TPOT）、并发、功耗、带宽、算力。
- 精度类：标准数据集准确率/得分（如 gsm8k）。
- 稳定性类：长压测期间告警、重启、宕机、关键日志异常计数。
- 兼容性类：编译通过率、样例运行成功率、关键功能可用性。

建议统一结果模板字段：

- 用例名称 / 用例编号
- 测试时间与执行人
- 环境信息（硬件、驱动、固件、框架版本）
- 命令与参数
- 原始日志路径
- 关键指标
- 结论（通过/不通过）
- 异常说明与复现步骤

## 5. 需求落地建议（对接当前平台）

- 按测试域建立分类：`chip_basic`、`training`、`inference`、`accuracy`、`ecosystem`、`video_codec`。
- 每个用例在平台中对应标准化 schema：目的、前置条件、步骤、指标、阈值、结果解析规则。
- 推理性能建议内置 SLO 校验器（TTFT/TPOT 阈值）。
- 统一结果入库与报表导出，支持按芯片型号、版本、框架、时间维度对比。
- 对稳定性/兼容性用例增加“证据文件”必填（日志、截图、报告附件）。
